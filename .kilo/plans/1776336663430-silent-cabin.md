# Plan: Implement ESLint untuk Pencegahan Kesalahan Sintaks & Enforce Clean Architecture dengan JSDoc

## Goal
Implementasi ESLint yang ketat dengan JSDoc untuk mencegah kesalahan sintaks, enforce arsitektur Clean Architecture, dan memastikan kode quality tinggi pada proyek UserTeleNode.

## Analisis Proyek Saat Ini
- **Bahasa**: JavaScript murni (CommonJS)
- **Struktur**: Sudah ada arsitektur lapisan (Domain, Application, Infrastructure, Presentation)
- **Masalah**: Bug sintaks yang baru-baru ini diperbaiki, tidak ada linting otomatis
- **Dependensi**: Node.js v22.14.0, telegram@2.26.22

## Keputusan Teknis
### Pilihan Pemeriksaan Tipe: JSDoc (bukan TypeScript)
**Alasan**:
- Tetap menggunakan JavaScript murni, tidak perlu migrasi file
- Lebih cepat implementasi, tidak ada setup build TypeScript
- Cukup untuk proyek medium size ini
- JSDoc memberikan typedef yang memadai untuk pencegahan bug null/undefined

**Batasan JSDoc**:
- Tidak ada type safety saat runtime
- Lebih rentan human error dalam komentar
- Tidak bisa enforce interface implementation secara statis
- Perlu plugin eslint-plugin-jsdoc untuk validasi

## Rencana Implementasi

### Phase 1: Setup ESLint Dasar
1. Install dependencies: eslint, eslint-plugin-jsdoc, eslint-config-standard
2. Buat .eslintrc.js dengan konfigurasi dasar
3. Setup rules untuk JavaScript modern

### Phase 2: Konfigurasi JSDoc
1. Aktifkan eslint-plugin-jsdoc
2. Setup rules untuk typedef, @param, @returns, @type
3. Panduan penulisan JSDoc konsisten

### Phase 3: Kontrak Antarmuka Arsitektur
1. Definisi interface untuk setiap lapisan:
   - Presentation/UI Layer
   - Application/Service Layer
   - Domain Layer
   - Infrastructure Layer
   - Adapter Layer
2. Contoh typedef JSDoc untuk setiap interface
3. Implementasi minimal yang menunjukkan interaksi

### Phase 4: Struktur Folder & Enforcement
1. Rancang struktur folder yang direkomendasikan
2. Setup eslint-plugin-boundaries untuk enforce arsitektur
3. Aturan untuk mencegah komunikasi lintas batas

### Phase 5: Integrasi CI & Skrip
1. Setup npm scripts untuk lint, fix, type-check
2. Konfigurasi GitHub Actions untuk lint pada push/PR
3. Mekanisme pre-commit hooks

### Phase 6: Migrasi Bertahap
1. Identifikasi file-file yang perlu typedef
2. Tambahkan kontrak interface secara bertahap
3. Aktifkan rules ESLint secara gradual
4. Refactor import untuk sesuai arsitektur

## Mekanisme Enforcement Arsitektur
Menggunakan eslint-plugin-boundaries:
- Presentation hanya boleh import Application
- Application boleh import Domain & Infrastructure
- Domain tidak boleh import layer lain
- Infrastructure boleh import Domain interfaces
- Adapter sebagai boundary untuk external dependencies

## Konfigurasi ESLint Lengkap
```js
module.exports = {
  extends: ['standard'],
  plugins: ['jsdoc'],
  parserOptions: {
    ecmaVersion: 2022,
    sourceType: 'module'
  },
  rules: {
    // JSDoc rules
    'jsdoc/require-description': 'error',
    'jsdoc/require-jsdoc': ['error', {
      require: {
        FunctionDeclaration: true,
        MethodDefinition: true,
        ClassDeclaration: true
      }
    }],
    'jsdoc/require-param': 'error',
    'jsdoc/require-param-description': 'error',
    'jsdoc/require-returns': 'error',
    'jsdoc/require-returns-description': 'error',
    // Syntax prevention
    'no-unused-vars': 'error',
    'no-undef': 'error',
    'no-console': 'warn'
  },
  settings: {
    jsdoc: {
      mode: 'typescript'
    }
  }
};
```
